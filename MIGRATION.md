# Migration: switch `quantecon-book-theme` to use this fixtures repo

This document describes the changes needed in
[`quantecon-book-theme`](https://github.com/QuantEcon/quantecon-book-theme) to
replace its dependency on `lecture-python-programming` (cloned fresh each CI
run) with this static fixtures repo.

## Goals

- **Stability**: visual snapshot diffs reflect theme changes only, not
  upstream lecture edits.
- **Speed**: fixtures site builds in ~30–60s vs ~3–5 min for
  `lecture-python-programming` (no conda env, no scientific Python stack, no
  notebook execution).
- **Coverage**: 12 synthetic pages + a growing real-world corpus exercise more
  theme surfaces than the lecture-python-programming subset can.
- **Single preview target**: Netlify deploys against fixtures, so reviewers
  see one curated gallery per PR.

## Files to change in `quantecon-book-theme`

| File | Change |
|------|--------|
| `.github/workflows/ci.yml` | Replace lecture clone + conda + jb build with fixtures clone + pip + jb build |
| `.github/workflows/update-snapshots.yml` | Same replacement, both `/update-new-snapshots` and `/update-snapshots` jobs |
| `.github/workflows/integration.yml` *(new)* | Keep `lecture-python-programming` build as a nightly non-blocking smoke test |
| `tox.ini` | `[testenv:visual]` and `[testenv:visual-update]` — replace lecture clone with fixtures clone |
| `playwright.config.ts` | Default `SITE_PATH` updated to fixtures build path |
| `tests/visual/theme.spec.ts` | Page list updated to match fixture file paths; expanded to cover new surfaces |
| `tests/visual/__snapshots__/` | Will be fully regenerated by `/update-snapshots` after cutover (one-time churn) |
| `tests/visual/README.md` | Update setup instructions to reference fixtures repo |

## Pin the fixtures SHA

The theme repo must reference a **specific commit** of this fixtures repo, not
`main`. Otherwise fixture authoring would unexpectedly break theme PRs.

In each workflow:

```yaml
- name: Checkout fixtures
  uses: actions/checkout@v6
  with:
    repository: QuantEcon/quantecon-book-theme-fixtures
    ref: <fixtures-sha>            # pin to a specific commit
    path: fixtures
```

Store the pinned SHA somewhere obvious (e.g., a single `FIXTURES_SHA` env var
near the top of each workflow) so bumping it is a one-line change.

## Workflow snippets

### Replacement for the visual test job in `ci.yml`

```yaml
jobs:
  visual:
    runs-on: ubuntu-latest
    env:
      FIXTURES_REPO: QuantEcon/quantecon-book-theme-fixtures
      FIXTURES_SHA: <pinned-sha>
    steps:
      - name: Checkout theme
        uses: actions/checkout@v6

      - name: Checkout fixtures
        uses: actions/checkout@v6
        with:
          repository: ${{ env.FIXTURES_REPO }}
          ref: ${{ env.FIXTURES_SHA }}
          path: fixtures

      - name: Setup Python
        uses: actions/setup-python@v6
        with:
          python-version: "3.13"
          cache: "pip"

      - name: Install fixtures build deps + this PR's theme
        run: |
          pip install -r fixtures/requirements.txt
          pip install .

      - name: Build fixtures site
        working-directory: fixtures
        run: jb build . --warningiserror

      - name: Setup Node.js
        uses: actions/setup-node@v6
        with:
          node-version: "20"
          cache: "npm"

      - name: Install Playwright
        run: |
          npm ci
          npx playwright install --with-deps chromium

      - name: Run Visual Regression Tests
        id: visual-tests
        run: npm run test:visual
        continue-on-error: true
        env:
          SITE_PATH: fixtures/_build/html

      # ... (existing upload + PR comment + fail-if-failed steps unchanged) ...

      - name: Preview Deploy to Netlify
        uses: nwtgck/actions-netlify@v3.0
        with:
          publish-dir: "fixtures/_build/html/"
          production-branch: main
          github-token: ${{ secrets.GITHUB_TOKEN }}
          deploy-message: "Preview Deploy from GitHub Actions"
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}
```

Note the removed steps:

- `git clone --branch quantecon-book-theme https://github.com/QuantEcon/lecture-python-programming`
- `conda-incubator/setup-miniconda@v4` and the conda environment file
- `cd lecture-python-programming && jb build lectures --path-output ./`

Net result: ~40 lines of workflow removed, conda dependency removed entirely
from visual-test path.

-    command: `python -m http.server 8000 --directory ${process.env.SITE_PATH || "lecture-python-programming/lectures/_build/html"}`,
+    command: `python -m http.server 8000 --directory ${process.env.SITE_PATH || "fixtures/_build/html"}`,

## `tests/visual/theme.spec.ts` rewrite

The current spec hard-codes lecture-python-programming page names
(`/getting_started.html`, `/numpy.html`, etc.). After cutover, these paths
change to match the fixture file layout:

```typescript
// Synthetic pages — one per theme surface
const syntheticPages = [
  { name: "typography",      path: "/synthetic/typography.html" },
  { name: "definition-lists", path: "/synthetic/definition-lists.html" },
  { name: "code-blocks",     path: "/synthetic/code-blocks.html" },
  { name: "math",            path: "/synthetic/math.html" },
  { name: "admonitions",     path: "/synthetic/admonitions.html" },
  { name: "exercises",       path: "/synthetic/exercises.html" },
  { name: "proofs",          path: "/synthetic/proofs.html" },
  { name: "tables",          path: "/synthetic/tables.html" },
  { name: "figures",         path: "/synthetic/figures.html" },
  { name: "toc-deep",        path: "/synthetic/toc-deep-nesting.html" },
  { name: "long-page",       path: "/synthetic/long-page.html" },
  { name: "cross-refs",      path: "/synthetic/cross-references.html" },
];

// Real-world pages — frozen lecture captures
const realWorldPages = [
  { name: "prob-matrix",     path: "/real-world/from-lecture-python/prob_matrix.html" },
];
```

The `hasPlots` distinction can go away — fixtures don't execute notebooks, so
all rendered output is deterministic. Every page can take a full-page
screenshot.

The f-string test should move from the lecture-python `names.html` to
`synthetic/code-blocks.html`, which includes f-string content by design.

## Cutover order

To avoid a flag-day switch, do the migration on a branch with both systems
running in parallel for one PR, then drop the old path:

1. **Land this fixtures repo** on `main` of `quantecon-book-theme-fixtures`.
   Capture the SHA.
2. **Open a theme PR** that:
   - Adds the new fixtures-based workflow steps alongside the existing
     lecture-python-programming steps (both run, both gate the PR).
   - Pins to the captured fixtures SHA.
3. **Confirm the new path works**:
   - Comment `/update-new-snapshots` to seed initial baselines under the new
     paths.
   - Verify Netlify preview deploys against the fixtures site.
   - Verify Playwright reports look reasonable.
4. **Remove the old path** in the same PR (or a follow-up): delete the
   lecture-python-programming clone + conda + jb build steps, delete the old
   snapshots from `tests/visual/__snapshots__/` that no longer match any
   spec.
5. **Add `integration.yml`** as a separate PR — nightly, non-blocking,
   doesn't gate anything.
6. **Update [`tests/visual/README.md`](../quantecon-book-theme/tests/visual/README.md)**
   to reference the fixtures repo.

## Rollback

If the fixtures path breaks something fundamental after cutover, the rollback
is small: revert the workflow + playwright.config.ts + theme.spec.ts changes
on a branch, and restore the previous snapshot directory from git history. No
data is lost — the lecture-python-programming integration path can be brought
back with a `git revert`.

## Snapshot churn

The cutover regenerates every snapshot. Expect a single noisy PR commit from
`/update-snapshots`. After that, snapshot diffs in subsequent PRs should be
small and attributable to specific theme changes.
